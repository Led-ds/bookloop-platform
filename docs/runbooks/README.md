# Runbooks

**Finalidade:** procedimentos operacionais objetivos para tarefas recorrentes e incidentes.

**Quando usar:** durante operação/plantão. Cada runbook deve ser executável "de cima a baixo".

**Como evoluir:** após cada incidente, atualize/crie o runbook correspondente (post-mortem leve).

- [`deploy.md`](deploy.md) — publicar/promover uma versão.
- [`rollback.md`](rollback.md) — reverter uma versão.
- [`incident-db-connection.md`](incident-db-connection.md) — backend não conecta no RDS.
