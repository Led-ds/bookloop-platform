# 📚 docs — Documentação como Código

**Finalidade:** fonte canônica de conhecimento da plataforma. Toda a *prosa* (arquitetura,
decisões, padrões, onboarding, operação) vive aqui, versionada junto do código.

**Quando usar:** ao entender o "porquê" das coisas, ao onboardar, ao registrar uma decisão,
ou ao operar um incidente.

**Como evoluir:** todo conteúdo entra por PR. Decisões relevantes viram ADR em `decisions/`.
Diagramas em `diagrams/` (preferir Mermaid/texto versionável a imagens binárias).

| Subpasta | Conteúdo |
|----------|----------|
| `architecture/` | Visão de arquitetura, topologia de deploy, rede. |
| `decisions/` | ADRs (Architecture Decision Records). |
| `diagrams/` | Diagramas (Mermaid/C4) versionáveis. |
| `standards/` | Padrões de engenharia (prosa). |
| `onboarding/` | Como começar a usar a plataforma. |
| `runbooks/` | Procedimentos operacionais e resposta a incidentes. |
